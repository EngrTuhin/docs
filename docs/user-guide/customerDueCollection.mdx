# When Should You Use Customer Due Collection?

Use this page when:

- Customer pays old due
- Customer pays partial due
- Customer pays full due
- You want to record customer collection
- You want customer ledger and due report to stay correct

---

# Customer Due Collection List

The Customer Due Collection page shows all saved due collection records.

You can see:

| Column | Meaning |
|---|---|

# Add Customer Due Collection

<Steps>
  <Step title="Open Customer Due Collection">
    From the left sidebar, go to **Sales Management** and click **Customer Due Collection**.
  </Step>

  <Step title="Click Add">
    Click the green **Add** button.
  </Step>

  <Step title="Select Warehouse / Shop">
    Select the warehouse or shop where the payment is received.
  </Step>

  <Step title="Search Customer or Retailer">
    Select the customer who is paying due.
  </Step>

  <Step title="Select Sales Person">
    Select sales person if needed.
  </Step>

  <Step title="Select Current Date">
    Select the payment date.
  </Step>

  <Step title="Enter Receipt No">
    Enter receipt number if you use manual receipt tracking.

    If not needed, you may keep it empty if the system allows.
  </Step>

  <Step title="Add Note">
    Add a short note if needed.

    Example: `Partial due payment received`
  </Step>

  <Step title="Select Payment Type">
    Select payment type.

    Example: `Cash`
  </Step>

  <Step title="Enter Due Payment">
    Enter the amount received from the customer.
  </Step>

  <Step title="Save Collection">
    Save the due collection record.
  </Step>
</Steps>

# Payment Type

Select how the customer is paying.

| Payment Type | Use |
|---|---|
| Cash | Payment received by cash |
| Bank | Payment received through bank |
| Mobile Bank | Payment received through mobile banking |

<Info>
Choose the correct payment type so cash, bank, and mobile banking reports stay correct.
</Info>

# Due Payment Section

The due payment section may include:

- Customer due amount
- Payment amount

# Full Due Collection Example

Use this when the customer pays the full due amount.

| Field | Example |
|---|---|
| Customer | `Rahim Store` |
| Previous Due | `5000` |
| Payment Type | `Cash` |
| Amount Collected | `5000` |
| Remaining Due | `0` |

---

# Partial Due Collection Example

Use this when the customer pays part of the due amount.

| Field | Example |
|---|---|
| Customer | `Rahim Store` |
| Previous Due | `5000` |
| Payment Type | `Cash` |
| Amount Collected | `2000` |
| Remaining Due | `3000` |

<Info>
If the customer pays partial due, the remaining balance will stay as customer due.
</Info>

---

# Receipt No

Receipt No is used to track payment records.

Use Receipt No when:

- You have a manual money receipt
- Customer wants receipt tracking
- Accounts team needs reference
- You want to match collection with another document

Example:

| Field | Example |
|---|---|
| Receipt No | `MR-1001` |
| Receipt No | `125021` |
| Receipt No | `CASH-2026-001` |

If you do not use receipt numbers, keep it empty if the system allows.

---

# Action Menu

Each collection record has an **Action** menu.

Common actions may include:

| Action | Use |
|---|---|
| View | See collection details |
| Print | Print receipt copy |
| Edit | Edit collection if permission allows |
| Delete | Delete collection if permission allows |

<Warning>
Edit or delete due collection carefully. It may affect customer due, ledger, cash/bank balance, and reports.
</Warning>

---

# How Due Collection Affects the System

Customer Due Collection may affect these areas:

| Area | Effect |
|---|---|
| Customer Due | Customer due balance decreases |
| Customer Ledger | Collection entry appears in ledger |
| Cash / Bank / Mobile | Selected payment account balance increases |
| Reports | Collection appears in collection reports |
| Sales Due | Related customer due is reduced |

---

# Customer Due Collection vs Sales Payment

| Option | When to Use |
|---|---|
| Sales Payment | When receiving payment during sales invoice |
| Customer Due Collection | When collecting old or previous due after invoice is already saved |

<Info>
If the invoice is already saved with due, collect later payment from **Customer Due Collection**.
</Info>

---

# Common Mistakes

| Mistake | Problem |
|---|---|
| Wrong customer selected | Due reduces for the wrong customer |
| Wrong amount entered | Customer balance becomes incorrect |
| Wrong payment type selected | Cash/bank/mobile report becomes wrong |
| Receipt number missing | Payment tracking becomes harder |
| Wrong date selected | Daily collection report becomes wrong |
| Duplicate collection entry | Customer due becomes incorrect |

---

# Best Practice

- Select the correct customer
- Check customer due before collection
- Enter the exact received amount
- Select the correct payment type
- Add receipt number if available
- Add note for partial payments
- Print receipt if customer needs it
- Check collection report at the end of the day
- Do not edit or delete collection without permission
