Koorri POS & Inventory System supports two types of POS sales:

<CardGroup cols={2}>
  <Card title="Regular POS" icon="shopping-cart">
    Use this for retail, wholesale, pharmacy, grocery, medicine shop, or normal product sales.
  </Card>

  <Card title="Restaurant POS" icon="utensils">
    Use this for restaurant table orders, kitchen print, customer print, and final bill.
  </Card>
</CardGroup>

<Info>
Use the POS type based on your business. Regular POS and Restaurant POS may show different options.
</Info>

# Choose POS Type

<Tabs>
  <Tab title="Regular POS">
    # Regular POS

    Regular POS is used for normal product sales.

    Use this for:

    - Retail shop
    - Wholesale shop
    - Pharmacy
    - Grocery shop
    - Medicine shop
    - General product sales

    ---

    ## Regular POS Workflow

    <Steps>
      <Step title="Open Sales Invoice">
        From the left menu, go to **Sales Management** and click **Sales Invoice**.
      </Step>

      <Step title="Search Product">
        Use the search box on the left side to find products.
      </Step>

      <Step title="Select Product">
        Click the product card to add it to the current order.
      </Step>

      <Step title="Select Customer">
        Select a customer from the right side customer section if needed.

        For cash sales, you may use a walk-in/default customer if your system supports it.
      </Step>

      <Step title="Check Quantity and Price">
        Make sure product quantity, price, and stock are correct.
      </Step>

      <Step title="Add Discount or VAT">
        Add discount or VAT only if needed.
      </Step>

      <Step title="Click Pay">
        Click **Pay** to open the payment page.
      </Step>

      <Step title="Process Payment">
        Enter payment details and click **Process**.
      </Step>

      <Step title="Print Invoice">
        Print the invoice for the customer if needed.
      </Step>
    </Steps>

    ---

    ## Regular POS Screen Options

    | Option | Use |
    |---|---|
    | Product Search | Search product by name |
    | Category Filter | Filter products by category |
    | Product Card | Click to add product to order |
    | Current Order | Shows selected products |
    | Customer | Select or add customer |
    | Discount | Add invoice discount |
    | VAT | Add VAT/tax |
    | Grand Total | Final payable amount |
    | Pay | Complete payment |
    | Hold | Keep order for later |
    | Draft | Save unfinished invoice |
    | Return | Process return from POS |
    | Clear | Clear current order |

    ---

    ## Regular POS Example

    | Field | Example |
    |---|---|
    | Customer | `Walk-in Customer` |
    | Product | `Savlon Hand Wash 250ml` |
    | Quantity | `2` |
    | Unit Price | `100` |
    | Discount | `0` |
    | VAT | `0` |
    | Grand Total | `200` |
    | Payment Status | `Paid` |

    ---

    ## Regular POS Notes

    <Warning>
    Before clicking Pay, always check product name, quantity, price, discount, VAT, customer, and grand total.
    </Warning>

    <Info>
    After completing the sale, stock and sales reports may update automatically.
    </Info>
  </Tab>

  <Tab title="Restaurant POS">
    # Restaurant POS

    Restaurant POS is used for table-based food orders.

    Use this for:

    - Restaurant
    - Cafe
    - Fast food shop
    - Table service business
    - Food court counter

    ---

    ## Restaurant POS Features

    | Feature | Use |
    |---|---|
    | Table Setup | Create restaurant tables |
    | Table Order | Take order for selected table |
    | Kitchen Print | Send order to kitchen |
    | Customer Print | Print customer copy before final payment |
    | Final Bill | Complete order and receive payment |
    | Hold Order | Keep running table order |
    | Draft | Save order without final payment |

    ---

    ## Step 1: Set Up Tables

    Before using Restaurant POS, create your tables.

    <Steps>
      <Step title="Open Table Setup">
        From the left menu, go to **Sales Management** and click **Table Setup**.
      </Step>

      <Step title="Click Add">
        Click the **Add** button.
      </Step>

      <Step title="Enter Table Name">
        Type the table name.

        Example: `Table 1`, `Table 2`, `VIP 1`, `Family Zone 1`
      </Step>

      <Step title="Save Table">
        Save the table.
      </Step>
    </Steps>

    <Info>
    Table Setup is needed only for restaurant or table-based sales.
    </Info>

    ---

    ## Step 2: Create Restaurant Order

    <Steps>
      <Step title="Open Sales Invoice">
        Go to **Sales Management → Sales Invoice**.
      </Step>

      <Step title="Select Table">
        Select the table for the customer order.
      </Step>

      <Step title="Select Customer">
        Select customer if needed.

        For normal restaurant orders, you may use walk-in customer if your system supports it.
      </Step>

      <Step title="Select Items">
        Search and click food items from the left product panel.
      </Step>

      <Step title="Check Current Order">
        Check item name, quantity, and price.
      </Step>

      <Step title="Kitchen Print">
        Print kitchen copy so the kitchen team can prepare the order.
      </Step>

      <Step title="Customer Print">
        Print customer copy if the customer wants to check the bill.
      </Step>

      <Step title="Final Bill">
        When the order is complete, click Pay and process payment.
      </Step>
    </Steps>

    ---

    ## Kitchen Print

    Kitchen Print is used to send the order to the kitchen.

    Kitchen copy may include:

    - Table name
    - Order number
    - Item name
    - Quantity
    - Order time
    - Waiter name if available
    - Special note if available

    <Info>
    Kitchen Print is not the final bill. It is only for food preparation.
    </Info>

    ---

    ## Customer Print

    Customer Print is used to give the customer a copy of the order or bill.

    Customer copy may include:

    - Item list
    - Quantity
    - Price
    - Discount
    - VAT
    - Grand total

    <Info>
    Customer Print can be used before final payment so the customer can check the bill.
    </Info>

    ---

    ## Final Bill

    Final Bill is created when the order is complete and payment is received.

    Final bill updates:

    - Sales report
    - Payment record
    - Customer due if any
    - Stock quantity if stock is enabled

    <Warning>
    Create the final bill only after checking all items, quantity, price, discount, VAT, and payment.
    </Warning>

    ---

    ## Restaurant POS Example

    | Field | Example |
    |---|---|
    | Table | `Table 3` |
    | Customer | `Walk-in Customer` |
    | Items | `Burger, Fried Rice, Soft Drink` |
    | Kitchen Print | `Yes` |
    | Customer Print | `Yes` |
    | Final Bill | `After payment` |
    | Payment Status | `Paid` |

    ---

    ## Restaurant POS Buttons

    | Button | Use |
    |---|---|
    | Kitchen Print | Send order to kitchen |
    | Customer Print | Print customer copy |
    | Pay | Complete final bill |
    | Hold | Keep table order running |
    | Draft | Save order without final payment |
    | Clear | Remove current order |
    | Return | Return product or item |

    ---

    ## When to Use Hold in Restaurant

    Use **Hold** when:

    - Customer is still sitting at the table
    - More items may be added later
    - Order is not ready for final bill
    - Payment is not received yet

    <Warning>
    Do not use Kitchen Print as the customer final bill.
    </Warning>
  </Tab>
</Tabs>

# Payment Page

After clicking **Pay**, the payment page will open.

Use this page to receive payment from the customer and complete the invoice.

The payment page has two main parts:

| Area | Use |
|---|---|
| Payments | Enter payment method, amount, account, reference, date, and notes |
| Summary | Check grand total, paid amount, and due amount |

---

# Payment Methods

You can select how the customer is paying.

| Method | Use |
|---|---|
| Cash | Payment received by cash |
| Bank | Payment received through bank |
| Mobile Bank | Payment received through mobile banking |

---

# Payment Fields

| Field | Use |
|---|---|
| Method | Select Cash, Bank, or Mobile Bank |
| Amount | Enter customer payment amount |
| Account / Wallet | Select bank account or mobile wallet if needed |
| Reference | Enter transaction ID, cheque number, slip number, or note |
| Date | Select payment date |
| Notes | Add payment note if needed |

---

# Payment Summary

The right side shows the invoice payment summary.

| Field | Meaning |
|---|---|
| Grand Total | Total invoice amount |
| Amount Given | Amount paid by customer |
| Paid | Amount received for this invoice |
| Due | Remaining unpaid amount |

---

# Receive Full Payment

Use this when the customer pays the full invoice amount.

<Steps>
  <Step title="Click Pay">
    After adding products to the invoice, click **Pay**.
  </Step>

  <Step title="Select Payment Method">
    Select `Cash`, `Bank`, or `Mobile Bank`.
  </Step>

  <Step title="Enter Full Amount">
    Enter the full invoice amount in the Amount field.
  </Step>

  <Step title="Check Summary">
    Make sure Due amount becomes `0`.
  </Step>

  <Step title="Click Process">
    Click **Process** to complete the payment and invoice.
  </Step>
</Steps>

---

|---|---|
| Grand Total | `1000` |
| Amount | `1000` |
| Paid | `1000` |
| Due | `0` |
| Payment Status | `Paid` |

# Receive Partial Payment

Use this when the customer pays only part of the invoice amount.

<Steps>
  <Step title="Click Pay">
    Open the payment page.
  </Step>

  <Step title="Select Payment Method">
    Select payment method.
  </Step>

  <Step title="Enter Paid Amount">
    Enter the amount received from customer.
  </Step>

  <Step title="Check Due">
    The system will show the remaining amount as Due.
  </Step>

  <Step title="Click Process">
    Click **Process** to save payment and invoice.
  </Step>
</Steps>

---

## Partial Payment Example

| Field | Example |
|---|---|
| Grand Total | `1000` |
| Amount | `600` |
| Paid | `600` |
| Due | `400` |
| Payment Status | `Due` or `Partial` |

<Info>
If the customer does not pay the full amount, the remaining balance will be customer due.
</Info>

# Add Multiple Payments

Use **Add another payment** when the customer pays using more than one method.

Example:

| Payment Method | Amount |
|---|---|
| Cash | `500` |
| Mobile Bank | `500` |

Total paid = `1000`

<Steps>
  <Step title="Enter First Payment">
    Select method and enter the first payment amount.
  </Step>

  <Step title="Click Add Another Payment">
    Click **Add another payment**.
  </Step>

  <Step title="Enter Second Payment">
    Select another method and enter the second payment amount.
  </Step>

  <Step title="Check Summary">
    Make sure total paid and due amount are correct.
  </Step>

  <Step title="Click Process">
    Click **Process** to complete the payment.
  </Step>
</Steps>

# Payment Page Common Mistakes

| Mistake | Problem |
|---|---|
| Wrong amount entered | Paid and due balance becomes wrong |
| Wrong payment method selected | Cash/bank/mobile report becomes wrong |
| Reference not added | Transaction tracking becomes difficult |
| Due not checked | Customer due may be incorrect |
| Process clicked too early | Wrong invoice may be saved |

# Hold Order

Hold Order is used when the order is not complete yet.

Use Hold when:

- Customer wants to add more products later
- Payment is not ready
- Restaurant table order is still running
- You want to pause the invoice

<Steps>
  <Step title="Add Products">
    Select products for the order.
  </Step>

  <Step title="Click Hold">
    Click **Hold**.
  </Step>

  <Step title="Resume Later">
    Open the held order later and complete payment.
  </Step>
</Steps>

---

# Draft Order

Draft is used to save an invoice without final payment.

Use Draft when:

- Invoice is not final
- Customer has not confirmed the order
- You need to review the invoice later
- Sales entry is incomplete

---

# Clear Order

Clear removes all selected products from the current order.

<Warning>
Use Clear carefully. It removes the current cart items from the screen.
</Warning>

# Best Practice

- Search product carefully
- Check stock before selling
- Select customer for due or credit sales
- Use walk-in/default customer only for cash sales if allowed
- Use kitchen print only for restaurant preparation
- Use customer print for customer review
- Use final bill only after payment confirmation
- Use hold order for running restaurant orders
- Use payment reference for bank or mobile banking
- Check daily sales report after closing sales
