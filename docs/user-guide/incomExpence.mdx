# Income & Expense

Income & Expense module is used to record, track, and manage all financial transactions including income, expenses, and other monetary activities of the business.

It helps businesses maintain accurate daily financial records, analyze spending patterns, and organize transactions by categories for better financial reporting.

---

# Purpose of Income & Expense

You use this module to:

- Record daily income and expenses
- Track all financial transactions with proper references
- Categorize expenses for better analysis
- View daily financial overview and summaries
- Maintain proper financial records for accounting and reporting

---

# Income & Expense Workflow Overview

Financial transactions generally follow this flow:

1. Add New Transaction (Income / Expense)
2. Select or Create Category
3. Enter Amount and Description
4. View and Filter Transactions
5. Manage Expense Categories (Setup)

---

# Income & Expense Modules

You will find the following modules under **Finance → Income & Expense**:

## 1. Transactions

Record and manage all income and expense entries.

## 2. Categories

Create and manage expense/income categories.

# Summary

Income & Expense provides a complete system for recording daily financial activities, tracking expenses by category, and generating quick financial insights through overview cards and detailed transaction history.

# 1. Transactions

Transactions is the primary module used to record, view, and manage all income and expense entries in the system.

It allows businesses to maintain a complete log of daily financial activities with proper categorization and reference numbers.

<Info>
  This is the main operational page under Income & Expense used for recording
  and viewing transactions.
</Info>

---

## Purpose

You use this module to:

- Record new expenses and income
- View daily financial overview
- Filter and search historical transactions
- Track amounts by category and type
- Maintain financial records for reporting

---

## Transactions Overview

The Transactions page shows a **Daily Overview** dashboard with key metrics and a detailed table of all entries.

### Daily Overview Cards

- Today's Entries
- Today's Income
- Today's Expense
- Today's Other
- Today's Net
- Filtered Entries, Filtered Amount, Avg Amount, Top Category, etc.

### Transaction Table Columns

| Column       | Description                                |
| ------------ | ------------------------------------------ |
| SL           | Serial number                              |
| Reference    | Transaction ID (e.g., EXP-0403-2026)       |
| Date         | Transaction date                           |
| Category     | Expense category (e.g., nasta, RIKSA VARA) |
| Amount       | Transaction amount                         |
| Type         | Income / Expense / Other                   |
| Description  | Additional notes                           |
| Sales Person | Assigned person (if any)                   |
| Action       | View/edit options                          |

---

## Add New Transaction (Expense Entries Form)

This form appears after clicking **+ Add** (or **Add Expense**).

---

## Header Fields

| Field       | Description                          |
| ----------- | ------------------------------------ |
| Date\*      | Transaction date (auto-filled)       |
| Category\*  | Select expense category              |
| Type\*      | Select type (Expense, Income, Other) |
| Amount\*    | Transaction amount                   |
| Description | Additional notes                     |

---

## Additional Controls

- **+ Add More** → Add multiple entries in one go
- **Type Wise Summary** → Shows Income, Expense, Other, Net, and Grand Total
- **Cancel** and **Save** buttons

---

## Buttons & Controls

### On Listing Page

- **+ Add** (Green Button) → Create new transaction
- Search → Find transactions
- Filters → Filter by date, category, type, etc.
- Analytics → View financial insights

### On Add Form

- Add More
- Cancel
- Save

---

## How It Works

### Step 1: Open Module

Go to: **Finance → Income & Expense → Transactions**

### Step 2: Add Transaction

1. Click **+ Add**
2. Enter/select Date
3. Choose Category
4. Select Type (Expense/Income)
5. Enter Amount
6. Add Description (optional)
7. Click **Save**

---

## After Saving

- Transaction appears in the main list with a unique reference number
- Daily overview cards update automatically
- Transaction is available for filtering and reporting

---

## Business Rules

- Date and Amount are required
- Category selection is mandatory
- Proper categorization helps in accurate financial reporting
- All transactions contribute to daily net calculation

---

## Example Use Case

A shop records daily expenses:

- Category: "nasta"
- Type: Expense
- Amount: 200
- Description: "Morning tea & snacks"
  The transaction is recorded with reference EXP-XXXX-2026 and reflected in today's expense total.

---

## Summary

Transactions is the core module for recording all financial activities. It provides both quick daily insights and detailed historical records.

# 2. Categories

Categories module is used to create and manage different types of income and expense categories.

It helps standardize transaction recording and enables better financial analysis by grouping similar expenses.

---

## Purpose

This module is used to:

- Create new expense/income categories
- Organize financial transactions
- Maintain active/inactive categories
- Improve reporting accuracy

---

## What the Page Shows

### Categories Listing

The main page displays all available categories in a table.

**Columns:**
| Column | Description |
|--------|-------------|
| SL | Serial number |
| Category | Category name |
| Type | Expense / Income / Other |
| Status | Active / Inactive |
| Description | Additional info |
| Action | Edit / Delete |

---

## Add New Category Form

This form appears after clicking **+ Add**.

**Fields:**

- **Category Name\*** — Enter category name
- **Category Type\*** — Dropdown (Expense, Income, Other)
- **Status\*** — Active / Inactive
- **Description** — Optional notes

**Buttons:**

- Cancel
- Save Product

---

## How It Works

### Creating a New Category

1. Go to **Finance → Income & Expense → Categories**
2. Click **+ Add**
3. Enter Category Name
4. Select Category Type
5. Set Status (usually Active)
6. Add Description (optional)
7. Click **Save Product**

---

## After Saving

- New category appears in the list
- It becomes available when adding new transactions
- Can be set to Inactive when no longer needed

---

## Business Rules

- Category Name must be unique and clear
- Type (Expense/Income) affects reporting
- Keep commonly used categories Active

---

## Example Use Case

Common categories:

- "nasta" (Expense)
- "RIKSA VARA" (Expense)
- "kurbani" (Expense)
- "Toyoba Eid Bonus" (Expense)

---

## Summary

Categories is a setup module that defines the structure for all financial transactions. Proper category management is essential for accurate expense tracking and financial reporting.
